* 方案 A（最推荐，开发成本最低）：电脑/NAS/服务器运行 MediaCrawler + 提供一个本地 HTTP API；App 通过同一 Wi‑Fi 调用
  * 数据流：App -> 你的局域网 API（关键词/平台/页数）-> MediaCrawler 抓取 -> API 返回结构化 JSON -> App 列表展示/交给 AI 总结
  * 优点：手机端轻；反爬/登录更好处理（扫码登录在电脑端完成）；更新维护简单
  * 难点：需要同网；要做 API 鉴权（至少 token）防止被误扫/滥用
* 方案 B（可用但有成本）：云端部署 MediaCrawler（VPS/容器）+ 你自己的账号登录态管理 + App 直连
  * 优点：不依赖局域网；随时可用
  * 难点：合规/账号风控/封禁风险更高；需要更完整的鉴权、审计、限流
